Takeaways

1. Make your documentation readable

  • Break up the text often
  • First 3-5 words of every paragraph are most important
  • 65-90 chars per line is a good guideline

2. Care about SEO

  • Meta description should describe the content
  • URL should describe the content
  • Every page is a landing page
  • Your documentation should always win
  • Etc etc... read the Google Guide

Making it better

  • Look at your product from a user/task-driven perspective
  • Users should "hire" things you control for most failures
  • Make error messages explain how to fix the problem
  • Better yet, solve the problem for the user
